[BUAA OS Lab2]内存管理

序

在工作能够完成的时限内，工作量会一直增加，直到所有可用时间都被填充为止。
——《经济学人》，1995

这是西里尔Cyril·诺斯古德Northcote·帕金森Parkinson提出的一句话。在 1958 年，帕金森把这句话扩充成一本书，名为《帕金森定律：对于进度的追求》，同时凝练出了著名的帕金森定律：

在预算之内，支出的需求会一直增加，直到所有资源被用完为止。

在计算机领域，这一定律同样适用。给 128 MB 可用内存，软件就占用 128 MB；给 16 GB 可用内存，软件就占用 16 GB。内存越来越大，但软件用掉的也越来越多，却仿佛没有本质上的飞跃。所以，只要帕金森定律仍然生效，无论我们有多大的内存，内存都是稀缺的资源，做好内存管理是每个开发者的本职工作。

页控制块

我们构建的这 npage 长度的数组，每个元素被称作一个“页控制块”，它和物理页是一一对应的。

具体来说，对于这个 struct Page 类型的数组 pages，pages[i] 表示从 0 开始计数的第 i 个结构体，对应第 i 个物理页面，因此我们可以得到以下换算关系：

page2ppn 意为 page to physical page number。逻辑很简单：用我已知的页控制块地址减掉数组首地址，就能得到这个页控制块对应的是第几个物理页[2]。

page2pa 意为 page to physical address。每个物理页都有固定的大小，当我们知道这是第几个物理页之后，只需要乘以这个大小，就能得到这个物理页的基地址。

接下来，如果 freemem == 0，就让它等于 end。freemem 是一个全局变量，它默认是 0。也就是说只要是第一次调用 alloc 函数，这个条件一定为真。

下一步就是对齐，这里用到了一个 ROUND 宏，宏的定义是一堆位运算看了都头大，不过一言以蔽之，就是让 freemem 向上取整，直到可以被 align 整除。

接着我们临时保存一下 freemem，因为我们马上要改变它的值：freemem = freemem + n：让 freemem 往后移动 n个字节。紧跟着判断一下有没有超出可用的内存空间，如果 PADDR(freemem) >= memsize，就抛出错误，终止运行。

提示

还记得什么是 memsize 吗？
memsize 是一个全局变量，保存最大可用的物理地址。之前说到，memsize = 65536 * 1024，可想而知如果不对 freemem 做些什么，它将永远大于 memsize。

PADDR 是一个宏，它的定义如下：

这个宏的作用很奇妙，我们还得从物理地址和虚拟地址开始说起。

我们作为用户，在编写程序的时候，用的是物理地址还是虚拟地址呢？答案是虚拟地址，物理地址是不能随便被我们这种外行访问到的，不然要操作系统这一个中间商干啥。但是问题来了，现在我们在学 OS，我们不是外行啊！OS 软件还没编写好，页表还没建立起来，MMU 还没发挥它的功效，这时候我们需要访问物理地址，就需要借助 MIPS 操作系统内存的布局了[3]。

刚才说到，freemem 是 0x80400000 加上 n 个字节，我们暂且不管它到底是多少，只知道它比 0x80400000 大一些。我们一查 MIPS 的内存布局图，就会发现它大概率在内存的 kseg0 段。

而 kseg0 段的虚拟地址，和物理地址之间有一个非常简单的映射关系：

VAkseg0−0x80000000=PA VA_{kseg0} - \text{0x80000000} = PA VAkseg0​−0x80000000=PA

因此我们只需要给 freemem 减掉这个 0x80000000 就可以得到对应的物理地址了！这下就可以和 memsize 进行大小比较了。

最后的最后，返回这片地址空间的首指针，这样就完成了地址空间的申请，并且让 pages 指向了这片地址空间。现在我们可以用诸如 pages[i] 之类的语句来取出第 i 个 Page 结构体了。

Thinking 2.4

  • ASID 的必要性

    1. 每个进程都认为自己拥有完整的虚拟内存空间，因此它们所用的虚拟地址往往会产生冲突。当切换进程时，如果 TLB 中残留的原来进程的表项，就可能导致访问的数据错误。使用 ASID 进行区分可以隔离不同进程的虚拟地址。
    2. 切换进程时，可以只更改 EntryHi 寄存器，而不用刷新整个 TLB。下次切换回来时，TLB 中还能保存一部分表项，加快访问速度。

  • 最大数量：28 = 256 个地址空间。

Thinking 2.5

  1. tlb_invalidate 调用 tlb_out。

  2. tlb_invalidate 删除指定的 TLB 旧表项。

  3. tlb_out 的解释如下：函数首先将 Entry_Hi 寄存器的值保存在 $t0 中，然后将 $a0（指定的 Key）写入。

然后根据这个 Key 去找旧表项，索引会存入 Index。

接下来取出这个索引，并和 0 比较：如果小于 0，则表示这个表项不存在，跳转 NO_SUCH_ENTRY 函数。

如果大于等于 0，则表项存在，往 Entry_Hi, Entry_Lo0 和 Entry_Lo1 中写入 0。

最后用 tlbwi 指令，将三个寄存器的值写入表项，实现清零。

Thinking 2.6

  1. 触发 TLB Miss 后进入 TLB 重填步骤：do_tlb_refill。

  2. 根据页目录和虚拟地址寻找页表项：page_lookup。

  3. 分配页面：page_alloc。

Thinking 2.7

  • X86 采用段页式的内存管理机制，MIPS 采用的是纯粹的页式管理。

  • X86 的地址空间划分是灵活的，而 MIPS 是固定的划分。

  • X86 的 TLB 由硬件自动填充，而 MIPS 由软件管理。
